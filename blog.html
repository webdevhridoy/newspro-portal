Interview Questions about ES6

var: Scope essentially means where these variables are available for use. var declarations are globally scoped or function/locally scoped.
- var variables can be re-declared and updated
- var variables are hoisted to the top of their scope and initialized with a value of undefined.

So,when a variable is taken into a function by the var keyword, that variable will be scoped to the function. What does that mean? That means then that the variable cannot be used outside the function. The function will then be its world or area, it can be used anywhere within the function by mixing as much as we like.

let: let is now preferred for variable declaration. It's no surprise as it comes as an improvement to var declarations. It also solves the problem with var that we just covered.

Const: This keyword is used when we want to specify the value of something. var is slightly different from let because its value cannot be changed. Can be declared only once.

Difference: Regular functions created using function declarations or expressions are 'constructible' and 'callable'. Since regular functions are constructors, they can be called using the 'new' keyword. However, arrow functions are only 'callable' and not constitutive. Thus, we will get a run-time error when trying to create a non-constructible arrow function using the new keyword.

forEach: forEach() allows us to loop through an array and access each item individually, just like a for loop does but in a nicer syntax.
map: As mentioned for .forEach() , if we want to iterate over an array, manipulate the value(s) somehow and return the final result, we need to use a .map().
filter: filter() does what it says on the tin, it filters the array into smaller ones based on whether each value passes/fails the filter condition..

Template literals are string literals allowing embedded expressions using backtick characters (`). We can use multi-line strings and string interpolation features with them. Formerly known as template strings.
